# Задание 3 подробнее

## Временная сложность

Сортировка выбором всегда выполняет n(n-1)/2 сравнений.

**Формула:** n(n-1)/2 = O(n²)

| Случай | Сложность | Пояснение |
|--------|-----------|-----------|
| Лучший | O(n²) | n(n-1)/2 сравнений, 0 перестановок |
| Средний | O(n²) | n(n-1)/2 сравнений, ~n перестановок |
| Худший | O(n²) | n(n-1)/2 сравнений, n-1 перестановок |

## Пространственная сложность

**O(1)** - сортировка "на месте", дополнительная память не требуется.

## Примеры

| n | Сравнений |
|---|-----------|
| 100 | 4,950 |
| 1000 | 499,500 |
| 10000 | 49,995,000 |

## Вывод

Алгоритм квадратичный, эффективен только для небольших массивов (до 1000 элементов).

# Задание 5 подробнее

## Принцип работы

Обычная версия:
- Находит минимум → ставит в начало

Двунаправленная версия:
- Находит минимум → ставит в начало
- Находит максимум → ставит в конец
- За один проход заполняет 2 позиции

## Сравнение (случайные данные, n=10000)

| Версия | Время |
|--------|-------|
| Обычная | 1.79 сек |
| Двунаправленная | 1.54 сек |

## Ускорение

Двунаправленная версия быстрее на 10-15%

## Почему быстрее?

- Проходов по массиву: n/2 вместо n
- Сравнений на проходе: 2 раза больше
- Общее число сравнений: примерно такое же
- Выигрыш за счёт меньшего количества итераций цикла
